我不喜欢写技术文章。说实话，这句话我想了很久才敢写下来。毕竟在程序员这个圈子里，写博客、做分享、输出技术内容，似乎是一种默认的正确。但对我来说，它从来不是一件自然而然的事。

很早之前，我也在 CSDN 上写过一些博客。那时候刚入行，每学会一个小功能、每搞懂一个算法，就迫不及待地记录下来。也陆陆续续有一些人看，偶尔收到几条评论，心里还挺高兴的。但现在回头看，那些内容无非是些小模块的实现、某个 API 的用法、几道算法的题解——**说是技术文章，其实更像是学生时代的课堂笔记**。

如今是 AI 的时代。遇到问题，打开 ChatGPT 或者 Claude，几秒钟就能得到一个能用的答案。谁会去翻一篇博客？更不用说那些写于五六年前、依赖早已过时的技术文章了。我自己都不看。遇到问题，要么查官方文档，要么直接问 AI，轻松解决。**技术博客这个品类，在 AI 出现之后，似乎正在失去它曾经的意义**。

---

## 七年，也没什么高深的技术

我工作七八年了。公司交代的任务，都能出色地完成。经手的项目也不算少，日均 QPS 达到百万级别的也有好几个，两百万、三百万的流量都稳稳扛住。但你要问我用了什么高深的技术，我恐怕答不上来。

一般用 **Laravel** 就能解决大部分问题。实在不行，套一个 **Laraman** 的插件，性能就能提升一大截。偶尔加一些缓存——Redis 嘛，谁不会用。但说实话，**我到现在基本上没用过所谓的 MQ、微服务这些听起来很高深的东西**。不是不想用，是真的用不到。或者说，招我进去的那些公司，业务场景根本不需要这些。

有时候逛技术社区，看到别人大谈消息队列的削峰填谷、微服务的服务治理、分布式事务的最终一致性，我也会觉得自己是不是落伍了。但转念一想，我做的项目不也跑得好好的吗？几百万的 QPS，也没出过什么大问题。**技术说到底是为业务服务的，不是用来炫技的**。这个道理，我花了七八年才真正明白。

## 语言的旅程

我学的第一门语言是 **Java**。大学里老老实实地啃完了《Java 编程思想》，以为自己毕业后会成为一个 Java 程序员。但出来找工作才发现，杭州的 Java 程序员实在是太多了，多到简历投出去像石沉大海。

所以我的第一份工作，是做工业自动化。那时候用 **C#** 完成了一个项目。说起来也挺有意思的——进公司之前我只学过1个月的**C#** 基础，但项目来了，硬着头皮上，边学边做，居然也顺利完成了。后来公司业务需要采购设备、做自动化测试，我又去学了 **LabVIEW**。这个工业自动化领域的图形化编程语言挺有趣的。拖拖拽拽就能搭出一个控制系统，跟写代码是完全不同的体验。

2021 年，我入职了一家小公司。那里有一位浙大博士用 **PHP** 写了一个项目，准备让我接手进行迭代和维护。于是我又开始学 PHP，学 **Laravel**。本来以为又要经历一段痛苦的适应期，结果发现——**学 PHP 根本没什么门槛，没什么需要专门记录的技术**。它的封装已经太完美了。

---

## 这可能就是 PHP 的魅力

加上 Laravel 框架本身的优雅设计，整个项目都写得恰到好处。那位博士有时候会为了方便、为了好写，刻意降低一些可能的性能。但绝对不会影响到项目崩盘，因为他清楚地知道客户的需求量有多大。基本上整个项目都没做什么特别的优化——接口来了，业务逻辑直接访问数据库，拿结果，返回。就是这么简单直接。

**这可能就是 PHP 的魅力吧。** 它不跟你讲什么设计模式、不跟你谈什么架构哲学，它就是把事情做完了。简单、直接、够用。

> 写 PHP 的时候，你不会觉得自己在"搞技术"，你就是在解决问题。这种感觉，反而让我很踏实。

后来前端也转给我了。当时用的是 **Vue 2.0 + Vant3**。那位博士直接写了一个全局的 CSS 文件，把字体大小、颜色、边距这些常用样式全部封装好，根本不需要去引用别人的 UI 组件库。整体熟悉以后，开发速度飞快。页面简洁、好看，功能齐全。我想，**好的代码大概就是这样——不需要多复杂，但用起来就是顺手**。

---

## 这样也挺好

我不喜欢写技术文章，大概是因为我从来就不是一个"技术狂热者"。我不追求最新的框架，不痴迷于最优的架构，也不热衷于在社区里讨论那些听起来很厉害的技术名词。我只是一个普通的程序员，用着简单的工具，做着能用的产品，完成着公司交给我的任务。

但这就够了吗？我想是的。七八年来，我做的项目跑得稳稳当当，业务方满意，老板满意，我也满意。那些我"没用过"的技术——MQ、微服务、分布式缓存、读写分离——它们存在，当然有它们存在的道理。但**不是每个项目都需要它们，也不是每个程序员都必须掌握它们**。

这篇"技术文章"它一点都不技术。它只是一个工作了七八年的程序员，对自己职业生涯的一点诚实记录。如果你也和我一样，不怎么喜欢写技术文章，也没什么高深的技术，但一直在认真做事——那你并不孤单。
